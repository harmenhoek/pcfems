{% if help_section == "activity" %} All the PCFEMS page requests are listed here. These include user requests and anonymous requests. Note that an image load is also counted as a page request. {% elif help_section == "user" %} There are 2 types of users: Staff and Normal:
  • Normal members can access and use the main functionality of the system (overviews of items and storage locations, viewing item details, assigning, flagging, logging) and can update basic item information (images, storage location, ...). Normal members don't have access to the manage interface.
  • Staff members can can do the above, but in addition have extra privileges that can be set with the Permissions. Without permissions staff members can only access the Dashboard in Manage and see the Overviews (Open flags, Assigned Items and Items under Warrenty).
Permissions
Permissions can be set to Staff members to allow these members extra privileges.
Possible permissions are:
  • is_admin A system admin can set global system preferences, such as nomenclature and default user settings.
  • is_itemmoderator An item moderator can add new items, update items fully, delete items, add new flags, categories and location, and export items to csv.
  • is_usermoderator A user moderator can add and modify users and change user permissions.
Permissions can only be set by a Staff member with the permission to moderate users (is_usermoderator). {% elif help_section == "location" %} Each item in EMS is linked to a location. Each location is build up out of 2 parts:
1. Lab
2. Setup or Cabinet
When an item is in use (assigned to a user) the location is Lab + Setup, when the item is in storage the location is Lab + Cabinet.

Below the Labs, Setups and Cabinets can be updated and added individually, EMS takes care of combining these.

It is currently not possible to delete Labs, Cabinets and Setups. {% elif help_section == 'export' %}
Print QR codes
Printing QR codes is a 2-step process: exporting the data from EMS followed by importing the data and printing the labels with the Dymo software.

  1. Exporting the data can be done in the manage-environment of EMS. Go to Admin → Manage → Export. Select the ID range of items you wish to export (e.g. 10-50, for labels PCF0010-PCF0050), and press Export. 2 csv-files will be exported named export tracking.csv and export notracking.csv.

  2. Open the Dymo software DYMO Connect for Desktop (tested on Windows 10 with DYMO Connect for Desktop 1.3.2). Open the template pcfems_labels_version20210204_tracking.dymo and select Import data in the top of the screen. Select the csv file export tracking.csv that was just exported, verify you selected the tracking file. Once imported make sure the label size shown in the bottom of the screen matches the labels currently in the printer, and press Print to print the labels.

  3. Repeat step 2 for the no-tracking items. Use export notracking.csv in combination with pcfems_labels_version20210204_notracking.dymo.

For example a multimeter might fit in the General electronics category, but is better binned in the Multimeters category. If you are sure an item might form a new category with other items, this category can be added in the manage environment. {% elif help_section == 'flag' %} The goal of flagging is show users something is wrong with the item and to notify a staff member. Examples of flags are ’Needs repair’, ’Needs to be tested’, ’Missing’ or ’Incorrect information’.
Flagging is done by selecting Flag on the item detail page. To give more details of the problem a comment can be added to the flag. Once flagged, a specific flag icon will show up with the item on the list view and the flag will appear on the item detail page.

Flags have an icon, that can be picked from the Font Awesome (FA) library. {% elif help_section == 'labs' %} The location of an item consists of two parts, which depend on the item status:

Setups are thus the locations where items are located that are in use. This could also be an office. {% elif help_section == 'settings' %} System-wide settings can be changed here. Note that due to caching some changes may take time to appear. {% elif help_section == 'check' %} To verify if the EMS is up-to-date, checks can be performed. Based on several criteria items get a high, medium or low priority for checking.

High priority checks for items:
  • That are lost (no use location and no storage location).
  • In use not scanned or checked >{{ global_preferences.check__check_highpriority_inweeks }} weeks.
  • In storage not scanned or checked >{{ global_preferences.check__check_highpriority_storage_inweeks }} weeks.
Medium priority checks for items:
  • In use not scanned or between {{ global_preferences.check__check_mediumpriority_inweeks }} and {{ global_preferences.check__check_highpriority_inweeks }} weeks ago.
  • In storage not scanned or between {{ global_preferences.check__check_mediumpriority_storage_inweeks }} and {{ global_preferences.check__check_highpriority_storage_inweeks }} weeks ago.
Low priority checks for items:
  • Without High or Medium Priority that are tracked.
These ranges can be changes in Settings.

An item location is verified by:
  • Pressing Verify.
  • Changing the user (by pressing Change) via this screen (not the standard item detail screen).
  • Returning the item to storage (by pressing Return to Storage) via this screen (not the standard item detail screen).
  • Scanning the item QR-code.

This feature was added in January 2022 (version 1.2).
